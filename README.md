# 🌱 HabitTracker — Консольный трекер полезных привычек

![Логотип привычек](https://img.freepik.com/free-vector/plant-flat-style_78370-6593.jpg?semt=ais_hybrid&w=740&q=80)

**HabitTracker** — это минималистичное консольное приложение для формирования и отслеживания полезных привычек. Никаких подписок, облаков и уведомлений — только вы, терминал и ваша сила воли.

**Цель проекта:** Помочь пользователю методично внедрять новые привычки, визуализируя прогресс в виде *«цепочек успеха»* (streaks). Проект вдохновлён методом **Джеймса Клира** из книги «Атомные привычки».

> 📖 *«Вы не поднимаетесь до уровня своих целей — вы падаете до уровня своих систем».*  
> — Джеймс Клир

## ✨ Ключевые возможности

Приложение предлагает **простой**, но ***мощный функционал*** для работы с привычками:

- 📝 **CRUD-операции:** Создание, просмотр, редактирование и удаление привычек.
- 🔥 **Отслеживание серий (Streaks):** Автоматический подсчёт дней без пропусков[^1].
- 📅 **Календарь активности:** Визуализация в консоли по дням (аналог GitHub Contributions).
- 🎨 **Цветной вывод:** Использование `colorama` для наглядности статуса выполнения.
- 💾 **Локальное хранение:** Все данные сохраняются в читаемом `.json` файле без СУБД.

## ⚙️ Установка и первый запуск

Следуйте этой ***нумерованной инструкции***, чтобы начать работу:

1. **Клонируйте репозиторий** с GitHub:
   `git clone https://github.com/username/habittracker.git`
2. **Создайте виртуальное окружение** Python:
   `python -m venv .venv`
3. **Активируйте окружение**:
   - Windows: `.venv\Scripts\activate`
   - Unix/MacOS: `source .venv/bin/activate`
4. **Установите необходимые зависимости**:
   `pip install -r requirements.txt`
5. **Запустите главный скрипт**:
   `python -m habittracker`

> ⚠️ **Важно:** Убедитесь, что у вас установлен Python версии `3.9` или выше. Проверить это можно командой `python --version`.

## 📊 Таблица команд интерфейса

В приложении доступен следующий набор команд. Таблица иллюстрирует использование **выравнивания** в Markdown.

| Команда | Синтаксис | Описание действия | Пример |
| :--- | :---: | :--- | ---: |
| `add` | `add "Название"` | Добавляет новую привычку в трекер | `add "Читать 20 минут"` |
| `check` | `check ID` | Отметить выполнение привычки **сегодня** | `check 3` |
| `list` | `list [all\|active]` | Показать список привычек | `list active` |
| `stats` | `stats ID` | Показать статистику и график серий | `stats 1` |
| `delete` | `delete ID` | **Безвозвратно** удалить привычку | `delete 2` |

## 💎 Фрагмент исходного кода

Ниже показан пример функции, отвечающей за расчёт текущей серии (`streak`). Демонстрируется блок кода на Python и инлайн-вставки `datetime` и `timedelta`.

```python
from datetime import date, timedelta

class Habit:
    """Модель данных для одной привычки."""
    
    def __init__(self, name: str):
        self.name = name
        self.checkmarks: list[date] = []
    
    def calculate_streak(self) -> int:
        """Вычисляет текущую серию непрерывных выполнений."""
        if not self.checkmarks:
            return 0
        
        streak = 0
        today = date.today()
        
        # Идём от сегодняшнего дня назад
        for i in range(len(self.checkmarks)):
            check_date = today - timedelta(days=i)
            if check_date in self.checkmarks:
                streak += 1
            else:
                break
        return streak
```

## Контрольные вопросы
 
1. **Чем отличается вложенная цитата от обычной?**
 
Обычная цитата начинается с одного >, вложенная — с двух и более (>>, >>>), что создаёт многоуровневый блок[^1].
 
---
 
2. **Как сделать сноску в Markdown и где она отображается?**
 
В тексте ставится [^ключ], а в конце документа — [^ключ]: текст сноски. Сноски отображаются внизу страницы.
 
---
 
3. **В чём разница между - [ ] и - [x] в списках задач?**
 
- [ ] — невыполненная задача, - [x] — выполненная (чекбокс с галочкой).
 
---
 
4. **Какие два способа вставки эмодзи вы знаете?**
 
    Кодовые последовательности (:smile: → 😄)
 
    Прямое копирование символов эмодзи (🔐, ✅, ⚡).
 
---
 
5. **Зачем нужны HTML-вставки? Приведите пример.**
 
HTML позволяет добавлять элементы, которых нет в Markdown.
Пример: <kbd>Ctrl</kbd> + <kbd>C</kbd> → отображается как клавиши.
 
---
 
6. **Почему важно использовать правильные имена для внутренних ссылок (якорей)?**
 
Якоря строятся из заголовков (пробелы → дефисы, буквы → нижний регистр). Если имя не совпадёт, ссылка будет вести в никуда.
 
---
 
7. **Что произойдёт, если в таблице окажется разное количество столбцов в разных строках?**
 
Таблица отрендерится с ошибками: лишние ячейки могут быть проигнорированы, а недостающие — останутся пустыми.
